# Матрица приоритетов

## Определение
Инструмент для определения приоритетов задач на основе различных критериев (ценность, усилия, срочность, важность).

## Типы матриц приоритетов

### 1. MoSCoW Matrix
Приоритизация по категориям:
- **Must have** - обязательно
- **Should have** - желательно
- **Could have** - можно
- **Won't have** - не будем делать

### 2. Value vs Effort Matrix
Оценка по ценности и усилиям:
- **Quick Wins** - высокая ценность, низкие усилия
- **Major Projects** - высокая ценность, высокие усилия
- **Fill-ins** - низкая ценность, низкие усилия
- **Time Sinks** - низкая ценность, высокие усилия

### 3. Impact vs Urgency Matrix (Eisenhower)
- **Важно + Срочно** - сделать немедленно
- **Важно + Не срочно** - запланировать
- **Не важно + Срочно** - делегировать
- **Не важно + Не срочно** - удалить

## MoSCoW Matrix

### Must Have (M)
- Критически важно для проекта
- Без этого проект не имеет смысла
- Обычно 60% задач

**Примеры:**
- Авторизация пользователей
- Основная функциональность
- Безопасность данных

### Should Have (S)
- Важно, но не критично
- Можно отложить, но желательно включить
- Обычно 20% задач

**Примеры:**
- Улучшения UX
- Дополнительные фильтры
- Экспорт данных

### Could Have (C)
- Было бы хорошо иметь
- Можно легко отложить
- Обычно 20% задач

**Примеры:**
- Темная тема
- Дополнительные уведомления
- Статистика использования

### Won't Have (W)
- Не входит в текущий релиз
- Может быть в будущем
- Явно исключено

**Примеры:**
- Функции для будущих версий
- Экспериментальные фичи

## Value vs Effort Matrix

```
                    EFFORT
                Низкие    Высокие
VALUE    Высокая  [1]      [2]
         Низкая   [3]      [4]
```

### 1. Quick Wins (Быстрые победы)
**Высокая ценность + Низкие усилия**

- Делать в первую очередь
- Быстрый результат
- Мотивация команды

**Стратегия:** Начать с этого

### 2. Major Projects (Крупные проекты)
**Высокая ценность + Высокие усилия**

- Важные долгосрочные задачи
- Требуют планирования
- Высокий ROI

**Стратегия:** Планировать и выделять ресурсы

### 3. Fill-ins (Заполнители)
**Низкая ценность + Низкие усилия**

- Можно делать в свободное время
- Не критично
- Могут улучшить опыт

**Стратегия:** Делать когда есть время

### 4. Time Sinks (Пожиратели времени)
**Низкая ценность + Высокие усилия**

- Избегать или пересмотреть
- Не стоит усилий
- Может быть упростить?

**Стратегия:** Избегать или упростить

## Impact vs Urgency Matrix (Eisenhower)

### Квадрант 1: Важно + Срочно
- Кризисы, дедлайны
- Делать немедленно
- Минимизировать такие задачи

### Квадрант 2: Важно + Не срочно
- Стратегическое планирование
- Проактивная работа
- Фокус здесь

### Квадрант 3: Не важно + Срочно
- Прерывания, отвлечения
- Делегировать
- Минимизировать

### Квадрант 4: Не важно + Не срочно
- Пустая трата времени
- Удалить
- Избегать

## Как использовать

### Шаг 1: Список задач
- Собери все задачи из бэклога
- Убедись, что задачи понятны

### Шаг 2: Оценка
- Оцени каждую задачу по критериям
- Используй данные (метрики, опросы)

### Шаг 3: Размещение
- Размести задачи на матрице
- Визуализируй

### Шаг 4: Приоритизация
- Определи порядок выполнения
- Учти зависимости

### Шаг 5: Коммуникация
- Поделись с командой
- Объясни приоритеты
