# Оценка задач

## Методы оценки

### Story Points
- Относительная оценка сложности
- Используется в Agile/Scrum
- Шкала Фибоначчи: 1, 2, 3, 5, 8, 13, 21
- Не привязаны к времени

### Planning Poker
- Техника оценки в команде
- Каждый оценивает независимо
- Обсуждение расхождений
- Консенсус команды

### T-Shirt Sizing
- Простая категоризация
- XS, S, M, L, XL
- Быстрая оценка
- Для начальной приоритизации

### Часы/Дни
- Абсолютная оценка времени
- Традиционный подход
- Может быть неточной
- Зависит от опыта оценщика

## Факторы для оценки

### Сложность
- Техническая сложность
- Неизвестные технологии
- Интеграции

### Объем работы
- Количество задач
- Размер функциональности
- Документация

### Неопределенность
- Неясные требования
- Риски
- Зависимости

### Опыт команды
- Знакомство с технологиями
- Опыт с подобными задачами
- Навыки команды

## Процесс оценки

### 1. Подготовка
- Убедиться, что задача понятна
- Разбить на подзадачи при необходимости
- Уточнить требования

### 2. Оценка
- Использовать выбранный метод
- Вовлечь команду
- Обсудить расхождения

### 3. Валидация
- Сравнить с похожими задачами
- Учесть исторические данные
- Проверить реалистичность

### 4. Документирование
- Зафиксировать оценку
- Задокументировать предположения
- Обновить при изменении

## Velocity

### Определение
- Количество story points, выполненных за спринт
- Используется для планирования
- Рассчитывается на основе истории

### Использование
- Планирование следующего спринта
- Прогнозирование релизов
- Выявление трендов

### Формула
```
Velocity = Сумма Story Points завершенных задач / Количество спринтов
```

## Практические советы

✅ Используй относительную оценку (Story Points)
✅ Вовлекай всю команду в оценку
✅ Учитывай исторические данные
✅ Регулярно пересматривай оценки
✅ Документируй предположения
✅ Не смешивай Story Points и часы
✅ Учись на прошлых оценках

## Типичные ошибки

❌ **Слишком оптимистичная оценка**
- Решение: Добавлять буфер на неожиданности

❌ **Оценка без команды**
- Решение: Вовлекать всех в Planning Poker

❌ **Игнорирование неопределенности**
- Решение: Учитывать риски в оценке

❌ **Смешивание Story Points и времени**
- Решение: Использовать только Story Points для относительной оценки
