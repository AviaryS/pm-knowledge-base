# План изучения PM с нуля

## Цель
Систематическое изучение Project Management от основ до продвинутых практик.

## Длительность: 12 недель

---

## Неделя 1-2: Основы Project Management

### Теория
- [ ] Что такое проект и проектное управление
- [ ] Жизненный цикл проекта
- [ ] Роль Project Manager
- [ ] Ключевые концепции (scope, time, cost, quality)

### Практика
- [ ] Изучить [[01 Методологии/Waterfall|Waterfall модель]]
- [ ] Создать первый Project Charter
- [ ] Определить стейкхолдеров проекта

### Результат
Понимание базовых концепций PM и жизненного цикла проекта.

---

## Неделя 3-4: Планирование проекта

### Теория
- [ ] Work Breakdown Structure (WBS)
- [ ] Оценка времени и ресурсов
- [ ] Создание расписания
- [ ] Управление бюджетом

### Практика
- [ ] Создать WBS для проекта
- [ ] Оценить задачи (story points, часы)
- [ ] Построить расписание (Gantt chart)
- [ ] Изучить [[02 Матрицы и инструменты/RACI Matrix|RACI матрицу]]

### Результат
Умение планировать проект и оценивать задачи.

---

## Неделя 5-6: Agile и Scrum

### Теория
- [ ] Изучить [[01 Методологии/Agile|Agile манифест]]
- [ ] Понять [[01 Методологии/Scrum|Scrum фреймворк]]
- [ ] Роли в Scrum (PO, SM, Team)
- [ ] События Scrum (Sprint, Planning, Daily, Review, Retro)

### Практика
- [ ] Участвовать в Daily Scrum
- [ ] Провести Sprint Planning
- [ ] Создать Product Backlog
- [ ] Изучить User Stories

### Результат
Понимание Agile и умение работать в Scrum команде.

---

## Неделя 7-8: Управление стейкхолдерами и коммуникациями

### Теория
- [ ] Идентификация стейкхолдеров
- [ ] [[02 Матрицы и инструменты/Stakeholder Matrix|Матрица стейкхолдеров]]
- [ ] Планирование коммуникаций
- [ ] Управление ожиданиями

### Практика
- [ ] Создать матрицу стейкхолдеров
- [ ] Разработать план коммуникаций
- [ ] Провести встречу со стейкхолдерами
- [ ] Изучить [[05 Практики и техники/Управление стейкхолдерами|практики управления]]

### Результат
Умение работать со стейкхолдерами и управлять коммуникациями.

---

## Неделя 9-10: Управление рисками

### Теория
- [ ] Идентификация рисков
- [ ] [[02 Матрицы и инструменты/Risk Matrix|Матрица рисков]]
- [ ] Стратегии управления рисками
- [ ] Мониторинг и контроль рисков

### Практика
- [ ] Провести мозговой штурм рисков
- [ ] Создать матрицу рисков
- [ ] Разработать планы митигации
- [ ] Изучить [[05 Практики и техники/Управление рисками|практики]]

### Результат
Умение идентифицировать и управлять рисками проекта.

---

## Неделя 11-12: Инструменты и техники

### Теория
- [ ] [[02 Матрицы и инструменты/Priority Matrix|Матрицы приоритетов]]
- [ ] [[02 Матрицы и инструменты/Impact Effort Matrix|Impact/Effort матрица]]
- [ ] Kanban доски
- [ ] Метрики и KPI

### Практика
- [ ] Использовать матрицы для приоритизации
- [ ] Настроить Kanban доску
- [ ] Отслеживать метрики (velocity, burndown)
- [ ] Провести ретроспективу

### Результат
Владение ключевыми инструментами и техниками PM.

---

- [ ] Знаю Agile и Scrum
- [ ] Управляю стейкхолдерами
- [ ] Управляю рисками
- [ ] Использую PM инструменты
