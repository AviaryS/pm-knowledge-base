# Kanban

## Определение
Kanban - метод управления работой, визуализирующий поток задач от начала до завершения.

## Принципы Kanban

1. **Визуализация работы**
   - Все задачи видны на доске
   - Понятно, где каждая задача находится

2. **Ограничение работы в процессе (WIP)**
   - Максимальное количество задач на каждом этапе
   - Предотвращает перегрузку команды

3. **Управление потоком**
   - Фокус на движении задач
   - Минимизация времени выполнения

4. **Явные правила процесса**
   - Четкие критерии перехода между этапами
   - Понятные всем правила

5. **Улучшение через обратную связь**
   - Регулярный анализ метрик
   - Постоянное улучшение процесса

## Kanban Board

### Типичные колонки:
- **Backlog** - задачи к выполнению
- **To Do** - запланированные задачи
- **In Progress** - задачи в работе
- **Review** - задачи на проверке
- **Done** - выполненные задачи

### WIP Limits
- Ограничение количества задач в каждой колонке
- Пример: "In Progress" - максимум 3 задачи

## Метрики Kanban

### Lead Time
- Время от создания задачи до завершения
- Цель: минимизировать

### Cycle Time
- Время от начала работы до завершения
- Цель: минимизировать

### Throughput
- Количество задач, завершенных за период
- Цель: максимизировать

## Преимущества
✅ Гибкость - нет фиксированных спринтов
✅ Визуальная прозрачность
✅ Фокус на потоке
✅ Легко начать использовать

## Недостатки
❌ Может быть менее структурированным
❌ Требует дисциплины для соблюдения WIP
❌ Меньше предсказуемости сроков

## Когда использовать
- Непрерывный поток работы
- Нужна гибкость в планировании
- Команда предпочитает меньше формальностей
- Поддержка существующих продуктов
