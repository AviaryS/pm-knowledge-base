# SAFe (Scaled Agile Framework)

## Определение
SAFe - фреймворк для масштабирования Agile практик на уровень всей организации. Помогает большим командам работать в Agile манере.

## Уровни SAFe

### Team Level
- Отдельные Agile команды
- Scrum/Kanban практики
- Sprint циклы

### Program Level
- Несколько команд работают вместе
- Agile Release Train (ART)
- Program Increment (PI) Planning

### Portfolio Level
- Стратегическое планирование
- Управление портфелем проектов
- Выравнивание с бизнес-целями

## Ключевые концепции

### Agile Release Train (ART)
- Группа команд, работающих вместе
- Общий backlog и цели
- Синхронизированные релизы

### Program Increment (PI)
- Период планирования (обычно 8-12 недель)
- PI Planning событие
- Определение целей и планов

### Features
- Крупные функциональные блоки
- Требуют работы нескольких команд
- Управляются на Program уровне

## События SAFe

### PI Planning
- Двухдневное событие
- Все команды ART вместе
- Планирование следующего PI

### System Demo
- Демонстрация работающего продукта
- Показ Features
- Обратная связь от стейкхолдеров

### Inspect & Adapt
- Ретроспектива на уровне программы
- Анализ проблем
- План улучшений

## Роли в SAFe

### Release Train Engineer (RTE)
- Аналог Scrum Master на Program уровне
- Координирует ART
- Устраняет препятствия

### Product Management
- Управляет Features
- Определяет приоритеты
- Работает с Product Owners

### System Architect
- Техническое видение
- Архитектурные решения
- Руководство командами

## Преимущества
✅ Масштабирование Agile на всю организацию
✅ Синхронизация команд
✅ Стратегическое выравнивание
✅ Структурированный подход

## Недостатки
❌ Сложность внедрения
❌ Требует много ресурсов
❌ Может быть слишком формальным
❌ Не подходит для маленьких организаций

## Когда использовать
- Большие организации (100+ человек)
- Множество команд
- Нужна синхронизация
- Стратегическое выравнивание
